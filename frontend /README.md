# AlphaForge Frontend

## Overview

The frontend of AlphaForge provides the user interface for interacting with the multi-agent algorithmic trading platform.

It allows users to:

- monitor market data
- view trading signals
- track portfolio performance
- observe agent activity
- monitor system status

The frontend communicates with the backend API to retrieve data and trigger system actions.

---

# Technology Stack

The frontend is built using modern web technologies.

Main technologies include:

- React
- TypeScript
- Vite (for development and build tooling)
- REST API communication with the backend

These technologies provide a fast and modular development experience.

---

# Features

The frontend dashboard provides several capabilities.

### Market Monitoring

Users can monitor:

- asset prices
- market trends
- trading signals

---

### Trading Signals

The dashboard displays signals generated by the strategy agents.

Example signals:

- BUY
- SELL
- HOLD

These signals are derived from multiple analysis agents.

---

### Portfolio Monitoring

Users can view:

- current holdings
- portfolio allocation
- performance metrics

---

### Agent Monitoring

The dashboard can display the status of system agents.

Examples:

- sentiment analysis agent
- news analysis agent
- strategy agent
- risk management agent

# Running the Frontend Locally

Navigate to the frontend directory.


cd frontend


Install dependencies.


npm install


Start the development server.


npm run dev


The frontend will start at:


http://localhost:5173


---

# Backend Integration

The frontend communicates with the backend API.

Default backend endpoint:


http://localhost:8000


API requests include:

- retrieving market data
- fetching trading signals
- retrieving portfolio information
- monitoring agent status

---

# Building for Production

To create a production build:


npm run build


The optimized production files will be generated in the `dist` folder.
