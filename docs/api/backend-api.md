# Backend API

## Overview

The backend API provides the interface between the frontend dashboard and the internal multi-agent trading system.

The API is responsible for:

- exposing system data to the frontend
- triggering agent workflows
- retrieving market information
- managing portfolio data
- reporting system status

The backend is implemented using **FastAPI**.

---

# API Architecture

The API layer sits between the frontend application and the backend agents.

Frontend Dashboard
↓
Backend API (FastAPI)
↓
Agent Orchestrator
↓
Trading Agents


The API does not directly implement trading logic.  
Instead, it forwards requests to the appropriate agents.

---

# Base API URL

Example base path:

/api/v1


All endpoints are versioned to ensure backward compatibility.

Example:

/api/v1/market
/api/v1/portfolio
/api/v1/agents


---

# Health Check Endpoint

Used to confirm that the backend service is running.

### Endpoint

GET /api/v1/health


### Example Response

{
"status": "ok",
"service": "alphaForge-backend"
}


---

# Agent Endpoints

These endpoints interact with the system's trading agents.

### Get Agent Status

GET /api/v1/agents


Returns the status of all active agents.

Example response:

[
{
"name": "sentiment_analysis_agent",
"status": "running"
},
{
"name": "market_data_analysis_agent",
"status": "running"
}
]


---

### Trigger Agent

Allows manual execution of an agent.

POST /api/v1/agents/{agent_name}/run


Example:

POST /api/v1/agents/sentiment_analysis_agent/run


---

# Market Data Endpoints

These endpoints expose market information used by the dashboard.

### Get Market Data

GET /api/v1/market


Example response:


{
"symbol": "AAPL",
"price": 185.24,
"volume": 1200000
}

"confidence": 0.82
}
]
